Memahami Konsep Specificity di CSS

Specificity adalah konsep penting dalam CSS yang menentukan aturan mana yang "menang" saat ada konflik antar selektor.

Apa Itu Specificity?

Setiap selektor memiliki tingkat kekhususan berbeda. Specificity membantu browser menentukan aturan mana yang diterapkan jika ada lebih dari satu aturan yang tersedia dan menyebabkan konflik.

  • Selektor elemen: p → bobot 1
  • Selektor class: .judul → bobot 10
  • Selektor ID: #konten → bobot 100

Studi Kasus #1: Selektor Sama

Contoh:

<p>Ini paragraf</p>
p {
  color: red;
}

p {
  color: green;
}

Warna akhirnya hijau, karena selektor sama dan yang terakhir diterapkan.

Studi Kasus #2: ID vs Elemen

<p id="p1">Ini paragraf</p>
#p1 {
  color: red;
}

p {
  color: green;
}

Warna akhirnya merah, karena ID (bobot 100) lebih spesifik daripada elemen (bobot 1).

Cara Menghitung Specificity

Menggunakan format: inline | ID | class | element

  • Inline style → 1000
  • ID → 100
  • Class / pseudo-class → 10
  • Element → 1

Contoh perbandingan:

ul#sarapan li { color: green; } → 1 ID + 2 elemen = 0-1-0-2 = 102
.favorit { color: red; } → 1 class = 0-0-1-0 = 10

Yang menang adalah ul#sarapan li karena lebih spesifik (102 > 10)

Studi Kasus #3: Kombinasi Selektor

<ul id="sarapan">
  <li class="favorit">Nasi Goreng</li>
  <li>Mie Goreng</li>
</ul>
ul#sarapan li {
  color: green;
}

.favorit {
  color: red;
}

Warna Nasi Goreng adalah hijau, karena:

  • ul#sarapan li = 102
  • .favorit = 10

Jika ingin tetap merah, gunakan selektor yang lebih spesifik:

ul#sarapan .favorit {
  color: red;
}

Sekarang specificity-nya adalah 112, dan aturan merah yang diterapkan.
